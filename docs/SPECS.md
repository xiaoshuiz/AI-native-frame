# 📚 Spec System Overview

## 🎯 Why specs exist in this repo

Specs reduce rework by aligning product, design, and engineering before implementation starts.
For frontend-heavy teams, this is especially useful for interaction-heavy features.

## 🧱 Core templates

- 📌 Product requirements
- 🧩 Feature spec
- 🛠️ Technical spec
- 🎨 UI/UX spec
- 🧪 Test plan
- 🚢 Release checklist

Template files are located in `specs/templates/`.

## 🔁 Working agreement

1. Start with requirements and feature scope.
2. Define technical and UX details.
3. Confirm test strategy and release gates.
4. Link all related specs in the PR.

## 📝 ADR relationship

If a decision changes architecture direction, create or update an ADR in `docs/adr/`.
